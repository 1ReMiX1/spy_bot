# Spy Bot (GameDAG) — бот + мини-приложение Telegram

Телеграм-бот с играми (Шпион, Мафия, Крокодил, Правда или Действие).
Игры открываются внутри Telegram как **мини-приложение (Web App)**.

## Структура
```
spy_bot/
├─ bot.py            ← САМ БОТ (запускается на хостинге, Python)
├─ requirements.txt  ← зависимости Python
├─ Procfile          ← команда запуска: python bot.py
├─ runtime.txt       ← версия Python
├─ .gitignore
├─ .env.example      ← шаблон переменных (без секретов)
├─ .env              ← твой токен для локального запуска (В GIT НЕ ПОПАДАЕТ)
└─ docs/             ← МИНИ-ПРИЛОЖЕНИЕ (сайт, раздаётся через GitHub Pages)
   ├─ index.html
   ├─ app.js
   ├─ data.js
   └─ style.css
```

## Главное правило
- **Бот (`bot.py`)** работает на хостинге (bothost.ru). Это Python.
- **Мини-приложение (`docs/`)** — это сайт. Его раздаёт GitHub Pages по HTTPS.
- Файл `docs/app.js` — ДЛЯ БРАУЗЕРА. Его НЕЛЬЗЯ запускать на сервере через Node.
  Если хостинг запускает его — значит указан неверный стартовый файл (нужен `python bot.py`).

## ЧАСТЬ 1. Залить в GitHub и включить сайт

1. Залей все файлы этого репозитория в `github.com/1ReMiX1/spy_bot` (ветка `main`).
   Файл `.env` сам не зальётся — это нормально и правильно.
2. На GitHub: **Settings → Pages**.
3. **Source**: Deploy from a branch. **Branch**: `main`, папка **`/docs`** → **Save**.
4. Через 1–2 минуты сайт будет по адресу:
   ```
   https://1remix1.github.io/spy_bot/
   ```
   Открой его в браузере — должно появиться меню с 4 играми.

## ЧАСТЬ 2. Настроить бота на bothost.ru

1. Стартовый файл / команда запуска должна быть:
   ```
   python bot.py
   ```
   (ИМЕННО Python-файл, НЕ app.js!)
2. Переменные окружения:
   - `TELEGRAM_BOT_TOKEN` — уже есть на хостинге, бот её читает.
   - `WEBAPP_URL` — добавь в «Пользовательские переменные»:
     ```
     WEBAPP_URL = https://1remix1.github.io/spy_bot
     ```
3. Убедись, что в `requirements.txt` есть `python-telegram-bot` (он уже там).
4. Перезапусти бота.

В логах должно появиться «🤖 Бот начинает получать обновления...», а не ошибка `window is not defined`.

> Если хостинг упорно запускает `app.js` — залей на хостинг **только** `bot.py` и `requirements.txt`
> (папка `docs/` нужна только GitHub Pages, на хостинге бота она не нужна).

## ЧАСТЬ 3. Проверка
Открой бота в Telegram → `/start` → нажми кнопку → игра откроется в мини-приложении.

---

## ⚠️ Безопасность токена
Твой токен был показан в переписке, поэтому **сбрось его**: BotFather → /mybots → бот → API Token → Revoke.
После сброса обнови `TELEGRAM_BOT_TOKEN` на хостинге и в своём локальном `.env`.
НИКОГДА не коммить токен в GitHub — `.gitignore` уже защищает файл `.env`.

## Локальный запуск (необязательно)
```bash
pip install -r requirements.txt
python bot.py          # токен и WEBAPP_URL возьмутся из .env
```
